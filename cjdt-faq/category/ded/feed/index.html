Which is correct? https://deseanlewis.github.io/cjdtfaq/cjdtfaq/cjdt-faq/there-are-differences-between-the-allowed-values-in-the-fdle-data-element-dictionaries-deds-and-the-xsds-which-is-correct Thu, 11 Jun 2020 19:00:07 +0000 https://deseanlewis.github.io/cjdtfaq/knowledge-base/1-overview/ Both documents are correct and should be used in combination. Differences in values are sometimes the result of illustrating user itnerface (UI) values versus coded/transmission values.

For example with the Domestic Violence (DV) flag: The Yes/No values from DED description/example is what a user would expect to see on a screen in a system, i.e. the UI value. However, the Guide for Use section of the DED shows the actual True/False boolean format type. Technicians should use the True/False values for XML report transmissions.

For code lists, it is different. The XML/XSD contains a superset of codes; every code value may not apply to a system determinded by the reporting requirement. For example, CJDT data is limited to adults (or juveniles treated as adults) while OBTS/CCH data also includes juveniles. The XML/XSD contains a superset of code values that can be used for both CJDT and OBTS/CCH report transmissions. Therefore values like “waived to adult court”, that only apply to juveniles, would be used in transmissions for OBTS/CCH but not in transmissions for a CJDT record.
